iSAQB CPSA‑F Trainer — Guide

Practice app (multiple choice) for preparing for the CPSA‑F certification
(Certified Professional for Software Architecture – Foundation Level).
Runs in the browser – via a web link (phone & PC) or as a local file (PC/Mac),
no installation and no account. Contains 206 independently written questions across all 6 chapters of the curriculum,
plus the official iSAQB example exam as a dress rehearsal. Interface in German or English (toggle DE/EN top right).

Version 1.8.0

1. Access & start

Two ways — pick the one that fits your device:

A) Via the web link (recommended · works everywhere: iPhone, Android, PC/Mac)

  • App: https://antoniotroiano.github.io/isaqb-lernapp/
  • This guide online: https://antoniotroiano.github.io/isaqb-lernapp/help_en.html

Open the app link in your browser. For app‑like quick access from your home screen:

  • iPhone (Safari): Share icon → “Add to Home Screen”
  • Android (Chrome): Menu ⋮ → “Add to Home screen”

An internet connection is needed the first time; after that the app runs in the browser.

B) Local file (PC/Mac only)

Double‑click the file isaqb-trainer.html to open it in your browser — done, fully offline.

⚠️ iPhone/iOS: A local HTML file (e.g. from iCloud/Files) cannot be launched in Safari. On the iPhone, use the web link (A) instead. (Whether Android allows opening a local file directly is device‑dependent — here too the web link is the safe route.)

Learning progress is stored locally in the browser (localStorage) and never leaves the device. It applies per device/browser and is not synchronized between the web link and the local file.

2. Modes & learning paths

Exam simulation Rehearse the real thing Like the real exam: 40 questions · 75‑min countdown, evaluation only at the end, points per official iSAQB scoring, 60 % to pass.
🔁 Repeat Retain long term Spaced repetition (Leitner): practises exactly the questions due today per the study plan. Correct → longer interval (1 → 3 → 7 → 16 → 35 days), wrong → soon again. The number on the button = questions due.

📚 Study material Understand, not just tick boxes Read up on curriculum topics per learning goal (organized by chapter, with search). Tip: from an answered question, tap the LG number → jumps straight to the matching section.

🌐 Language: The DE/EN toggle at the top right switches the entire app between German and English — interface, all questions, study material, concept map and the official mock exam. Your choice is saved.

3. Features at a glance

  • Filter by chapter (1–6) and exam relevance (R1/R2/R3); a slider for the number of questions per round (also shows how many are available in the current filter).
  • Only answered wrong – targeted repetition of your mistakes (shrinks as you learn).
  • Bookmarks only – practises just the flagged questions (star at the top right of a question).
  • Prefer mistakes & Shuffle answers – weak questions more often, and shuffled answer order.
  • Fewer repeats – recently shown questions are avoided in following rounds.
  • Pause / Resume – an interrupted round is saved and resumed exactly where you left off.
  • Review answers – after each round, all wrong / partially correct questions with your choice, the solution and the explanation.
  • Bookmarks, dark mode (🌓 top right) and reset progress.
  • German/English – toggle (DE/EN) at the top right; switches the whole interface, the questions and the study material.

Searching the study material 📚

In the 📚 Study material tab you can use the search box at the top flexibly — it filters live as you type and ignores case:

  • By learning‑goal number: LG 03-08, lg 03-08 or simply 03-08.
  • By keyword/topic: searches titles and explanatory text, e.g. coupling, ATAM, microservices, interface.
  • By chapter: Chapter 3 or a word from the chapter title (e.g. design) shows the whole chapter.
  • Cross‑references included: when you search a LG number, learning goals that refer to it also appear (e.g. LG 03-08 additionally shows 03‑06 with “see LG 03‑08”).

Matches are expanded automatically. Clear the search box → back to the full, chapter‑grouped overview (tapping a learning goal toggles the text).

Tip: After an answered question you can tap the LG number directly in the feedback (e.g. LG 03-08) — it jumps straight to the matching study‑material section, no search needed.

Statistics 📊

  • Exam‑readiness estimate (accuracy on R1/R2 relative to the 60 % mark).
  • Coverage (seen / mastered – across all questions).
  • Accuracy by exam relevance (R1/R2/R3), question type (A/P/K) and chapter – with coloured bars.
  • Weakest chapters with a direct “practice” button.
  • Exam history – past simulations (date, percentage, passed/not).

Backing up data 💾

  • Export/Import (button in the statistics): back up your progress as text or a .json file and load it on another device/browser. Handy because the progress lives locally in the browser and is not synchronized between devices automatically.

4. Question types & scoring (in exam mode)

Every question uses checkboxes and states in its header how many answers to choose and how many points it is worth (1–2). Scoring follows the official iSAQB examination rules:

Type Meaning Scoring
A “Single Correct Answer” – one correct answer Correct = full points. Nothing / wrong / too many ticked = 0.
P “Pick‑from‑many” – N correct (“Select N answers”) Each correct tick +points/N, each wrong one −points/N; never below 0; more than N ticked = 0.
K “Choose Category” – assignment, one of 2 categories per row Each correct row +points/m, each wrong one −points/m (m = rows); never below 0.

Passing: at least 60 % of the total achievable points.
(Time: 75 min for native speakers, 90 min for non‑native speakers in the real exam.)

5. Scoring examples (how points are awarded)

Basic principle: correct ticks give proportional points, wrong ticks deduct proportionally – never below 0 per question. Points per question: 1–2 (shown in the question header). “Left out” (empty) deducts nothing; only wrong ticks cost.

A question — example: 2 points, exactly 1 correct

Answer Points
no selection 0
the correct one ticked 2
a wrong one ticked 0
several ticked 0

P question — example: 2 points, 2 of 4 correct (→ each correct = 1 point)

Answer Calculation Points
no selection – 0
both correct ones 1 + 1 2
1 correct + 1 wrong 1 − 1 0
only 1 correct (1 left out) 1 1
both wrong ones −1 − 1 → min 0 0
more than 2 ticked – 0

P question — example: 1 point, 4 correct (→ each correct = 0.25)

Answer Calculation Points
4 correct 4 × 0.25 1
3 correct + 1 wrong 3×0.25 − 0.25 0.5
2 correct + 2 wrong 0.5 − 0.5 0
1 correct + 3 wrong 0.25 − 0.75 → min 0 0
2 correct, 2 left out 2 × 0.25 0.5

K question (assignment) — example: 1 point, 3 rows (→ each correct row = 1/3)

Assignment Calculation Points
all 3 correct 3 × 1/3 1
no assignment – 0
2 correct + 1 wrong 2/3 − 1/3 1/3
1 correct + 2 wrong 1/3 − 2/3 → min 0 0
2 correct, 1 row empty 2 × 1/3 2/3

Key points

  • Ticking more than required (A: more than 1 · P: more than N) = 0 points for the whole question.
  • A question can never go negative (minimum 0) – mistakes in one question do not deduct points from other questions.
  • Passing: ≥ 60 % of the sum of all achievable points.

9. Technology & privacy (in brief)

  • A self‑contained HTML file (HTML + CSS + JavaScript), no external dependencies, questions stored as data directly in the code.
  • The app itself communicates with no server (no trackers, no data transfer). With the web link, the page is loaded once from GitHub Pages, after which everything runs in the browser; the local file works entirely without internet.
  • Progress lives solely in the browser’s localStorage. Clearing browser data (or “reset progress”) = start over from zero.
  • Adding more questions: new entries follow the data schema in the code
    ({ id, chapter, lz, r, type, q, options, correct, explanation }, or for K questions categories + rows).
